# Rule-based Knowledge Graph Completion with Canonical Models

Environment

1. create environment `conda create -n env python=3.8`
2. `pip install -r requirements.txt`
3. `git clone https://github.com/uma-pi1/kge.git`
4. `cd kge`
5. `git checkout a9ecd249ec2d205df59287f64553a1536add4a43`
6. `pip install -e .`

Workflow

1. Learn rules using AnyBURL (https://web.informatik.uni-mannheim.de/AnyBURL/)
2. Create explanations using files in `/explanations`: `source process.sh` (modify the paths in the process.sh)
3. Create datasets using `create_datasets.py` (Run `create_datasets.py -h` to get meanings of parameters)
4. Run aggregation script `aggregation.py` (Run `aggregation.py -h` to get meanings of parameters)


The zip named aggregation-code.zip contains the code as well as the documention of it.

Every other dataset zip file contains:
- The KGC dataset which can be put in the /data folder of the libkge python library
- Rule-sets used in paper (found under /rules)
- Explanation files generated by applying the rules to the knowledge graph (found under /expl)
- Datasets used to fit the aggregation model for each relation (found under /datasets)
